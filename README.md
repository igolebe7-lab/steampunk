# Паровая логистика

Колониально-индустриальная стимпанк-стратегия на гексах, создаваемая на Godot 4. Существующий механический прототип проверил fixed-tick симуляцию, физическую логистику, складские потоки, дороги, локальную пару и водопровод. После плейтеста короткого сценария проект переориентирован на полноценную непрерывную колонию с населением, физическим строительством, производственными цепочками, технологиями и развитием транспорта.

## Требования

- macOS;
- Godot 4.6.2 в `/Applications/Godot.app`.

Путь к Godot можно переопределить переменной окружения `GODOT_BIN`.

## Проверка

```bash
./scripts/check_project.sh
```

Команда импортирует ресурсы, запускает все тесты и выполняет короткий smoke-запуск главной сцены. Проверка намеренно нагружает один CPU-поток строгими replay-хэшами; обычная игровая сцена использует облегчённый runtime и не выполняет полный SHA-256 после каждого тика.

## Запуск редактора

```bash
'/Applications/Godot.app/Contents/MacOS/Godot' --editor --path .
```

После открытия проекта нажмите `F5`, чтобы запустить проект. `F6` запускает только открытую текущую сцену.

Игра ограничена 30 кадрами/с; на паузе включается щадящий режим 10 кадров/с. Скорость симуляции ×1/×2/×4 изменяет число fixed ticks, а не частоту отрисовки.

## Инструментированный плейтест

Техническая или пользовательская сессия запускается отдельной командой:

```bash
./scripts/run_playtest.sh PT-TECH
```

Идентификатор состоит из 1–32 латинских букв, цифр, `_` и `-`. Скрипт добавляет текущий короткий Git SHA. При старте Godot печатает `PLAYTEST_OUTPUT=<путь>` к локальному каталогу `user://playtests/`. После завершения там появляются `<ID>.json` и русский `<ID>-report.ru.md`.

Обычный запуск через `F5` или без аргумента `--playtest-session` не создаёт recorder. Raw JSON не следует коммитить до ручной проверки; сводка пользовательских сессий должна быть обезличена. Протокол наблюдателя находится в `docs/playtests/observer-guide.md`.

## Управление

- левая кнопка мыши — выбрать работника, здание, связь или гекс;
- средняя кнопка мыши и перетаскивание — переместить камеру;
- колёсико мыши — изменить масштаб в пределах `0.5…2.0`;
- `1`/`2`/`3`/`4` — осмотр, дорога, склад и грузовая связь;
- `5`/`6` — строительство и разбор водопровода;
- `Space` — пауза, `Escape` — отмена инструмента;
- верхние кнопки — пауза и скорости ×1/×2/×4;
- левые переключатели — связи, маршруты и загрузка.

Дорога и склад строятся за древесину. Для ручной связи сначала выберите исходное, затем целевое здание. Для трубы выберите непрерывный путь от насосной к котлу и повторно нажмите кнопку инструмента. Правая панель показывает параметры выбранного объекта и измеренную причину ожидания.

## Язык и локализация

Исходный язык игры и документации — русский. Пользовательские строки находятся в `localization/game.csv` и вызываются в коде по семантическим ключам через `tr()`. Английский столбец уже подключён как проверка возможности локализации; следующий язык добавляется новым столбцом CSV без изменения игрового кода.

Технические имена файлов, классов и методов остаются английскими, чтобы проект соответствовал принятым соглашениям Godot и GDScript.

## Структура проекта

- `scenes/` — игровые сцены;
- `src/app/` — сборка приложения и связь подсистем;
- `src/simulation/definitions/` — типы авторских данных;
- `src/simulation/loading/` — атомарная загрузка сценариев;
- `src/simulation/model/` — независимое от отображения состояние мира;
- `src/simulation/pathfinding/` — детерминированный A* и клетки взаимодействия;
- `src/simulation/commands/` — типизированные команды и стабильная очередь;
- `src/simulation/systems/` — логистический pipeline, применение команд и контроль инвариантов;
- `src/simulation/determinism/` — канонизация и SHA-256 состояния;
- `src/simulation/runner/` — выполнение фиксированных тиков;
- `src/presentation/world/` — геометрия, отрисовка и камера;
- `data/` — каталоги ресурсов, зданий и сценариев Godot Resource;
- `localization/` — каталог переводов;
- `tests/` — unit- и интеграционные тесты;
- `scripts/check_project.sh` — единая автоматическая проверка.
- `src/playtest/` — изолированная запись, анализ и отчёт плейтест-сессии;
- `docs/playtests/` — русские инструкции и шаблоны живого тестирования.

## Текущее состояние

Технические этапы 1–5 завершили первый вертикальный срез. Плейтест показал, что подготовленный короткий сценарий не содержит достаточного пространства для проверки настоящей стратегии: он автоматически приходит к первому удару молота и не проверяет исследование карты, основание поселения, население, физическое строительство и развитие отраслей.

Серия пользовательских тестов старого сценария остановлена. Прототип сохраняется как технический полигон, но больше не определяет структуру игры.

Новая нормативная цель описана в русском мастер-GDD. Следующая рабочая версия должна начинаться с готовой экспедиции на процедурной карте и включать полноценный цикл эпох I–II: домохозяйства, еду, воду, жильё, добычу, строительство, склады, производство и физические исследовательские наборы. Новый план реализации составляется только после проверки мастер-документа.
